What AutoRNAseq is

AutoRNAseq is a local, Dockerized RNA-seq analysis environment that takes the user from raw sequencing files to biological interpretation. It is built for people who want a guided interface, repeatable outputs, and a private workflow that runs on their own machine instead of a public server.

Input

FASTQ files, SRA accessions, metadata tables, reference genomes, and annotation files.

Core analysis

Quality control, trimming, alignment, gene counting, and DESeq2 differential expression analysis.

Biological interpretation

GO enrichment, KEGG pathway analysis, GSEA, STRINGdb network analysis, and ML-based biomarker ranking.

Outputs

Downloadable tables, publication-ready plots, network views, and report-friendly summaries.

What each step produces

Each module gives concrete outputs so the user can inspect quality, compare samples, and move into interpretation with confidence.

Stage 01

Quality Control

Rqc, FastQC, and MultiQC-style summaries help users spot base-quality drops, adapter issues, or unusual read composition before downstream analysis begins.

Outputs: quality tables, per-cycle plots, summary dashboards.
Stage 02

Filtering and Trimming

Reads can be cleaned with fastp or Trimmomatic, producing trimmed FASTQ files and improved readiness for alignment.

Outputs: trimmed reads, before/after QC comparison, length-filtered files.
Stage 03

Alignment and Quantification

Reference-based alignment and featureCounts summarization create BAM files and gene-level count matrices that form the basis of expression testing.

Outputs: BAMs, mapping summaries, count tables.
Stage 04

Differential Expression

DESeq2 produces contrast-ready gene tables that can be ranked by significance and fold change for downstream biology.

Outputs: DEG tables, volcano plots, top-gene summaries.
Stage 05

Functional Interpretation

GO, KEGG, and GSEA modules translate the gene list into pathway-level and process-level insight using enrichment methods.

Outputs: enrichment tables, dot plots, bar plots, maps, leading-edge genes.
Stage 06

Networks and Biomarkers

STRINGdb interaction networks and ML ranking help surface hub genes and candidate biomarkers for follow-up validation.

Outputs: hub tables, centrality plots, ROC curves, feature importance tables.

Quick start for non-coders

The shortest route from download to results.

  1. Download the latest release from GitHub.
  2. Install Docker Desktop if it is not already installed.
  3. Run the Windows or macOS/Linux startup script from the release package.
  4. Open http://localhost:3838 in your browser.
  5. Upload FASTQ files and metadata, then follow the sidebar modules.

Workflow depth

Each stage of the RNA-seq workflow is accessible in its own module, with inputs, controls, outputs, and help notes arranged in the same browser session.

Result types

Tables, plots, interactive networks, ranked gene lists, biomarker scores, and report-ready downloads are generated from the same workflow outputs.

Who it helps

Wet-lab researchers, students, and labs that want a guided transcriptomics workflow without maintaining their own bioinformatics server stack.
